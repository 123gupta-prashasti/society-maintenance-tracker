# Society Maintenance & SLA Tracker

An interactive, premium, full-stack platform for apartment cooperative societies to lodge and manage maintenance complaints, track status history, pin critical notices, and verify simulated outbound email logs.

## 🚀 Live Demonstration Highlights

- **Reviewer Hot-Login Panel:** No registration or typing required to start testing! On the login page, you can choose between pre-seeded test profiles:
  - **Sarah Jenkins (Admin)**
  - **John Doe (Resident)**
  - **Alice Smith (Resident)**
- **Role-Based Provisioning:** Enter any email address. If it contains the word `"admin"`, you will be granted administrative access; otherwise, you will be registered as a resident.
- **Time-Simulator Slider:** When filing a complaint, slide the helper to backdate ticket creation by up to 10 days to instantly trigger the overdue warnings on the admin dashboard.
- **Configurable SLA Threshold:** Admins can adjust the overdue threshold from 3 days to 1 day in real-time, instantly escalating pending tickets.
- **Simulated SMTP Outbox Logs:** View exact outbound emails generated by notice broadcasts or ticket status updates.

---

## 🛠️ Project Architecture

Built using a modern, unified full-stack architecture:
1. **Frontend SPA:** React 19, Tailwind CSS (v4), and custom-engineered lightweight components.
2. **Backend Server:** Express 4 executing with `tsx` (TypeScript execute) and bundled with `esbuild`.
3. **Database:** MongoDB Atlas (via Mongoose) for persistent, durable cloud data storage.
4. **Cloud Media Storage:** Cloudinary CDN for secure image upload and hosting.

## 📦 Local Setup Instructions

Follow these commands to clone, install, and execute the server locally on your workstation:

### 1. Install dependencies
```bash
npm install
```

### 2. Start Full-Stack Server (Development Mode)
```bash
npm run dev
```
The server will start running on **http://localhost:3000** with hot-reload asset parsing.

### 3. Compile Production Bundle
```bash
npm run build
```
This builds static client bundle folders inside `dist/` and compiles `server.ts` into a standalone, optimized CommonJS file `dist/server.cjs`.

### 4. Run Production Server
```bash
npm run start
```

## 🗄️ Database Schemas

### User Collection (MongoDB Atlas -> `users`)
Holds residents and administrative committee accounts:
```typescript
interface User {
  id: string;        // "usr_xxxxx" (Primary Key)
  email: string;     // Unique identifier
  name: string;      // Profile display name
  role: 'resident' | 'admin';
  apartment: string; // E.g. "Block A - 402" or "Admin Office"
  createdAt: string; // ISO 8601 string
}
```

### Complaint Collection (MongoDB Atlas -> `complaints`)
Saves lodging requests, photo attachments (stored as Cloudinary CDN URLs), and embeds status history records:
```typescript
interface Complaint {
  id: string;              // "cmp_xxxxx" (Primary Key)
  residentId: string;      // Foreign Key -> User.id
  residentName: string;
  residentApartment: string;
  category: string;        // "Plumbing", "Electrical", "Elevator", etc.
  description: string;
  photoUrl?: string;       // Cloudinary CDN secure URL
  status: 'Open' | 'In Progress' | 'Resolved';
  priority: 'Low' | 'Medium' | 'High';
  createdAt: string;
  updatedAt: string;
  overdue: boolean;        // Computed based on threshold settings
  history: ComplaintHistory[]; // Embed timeline
}
```

### ComplaintHistory Relation (embedded in `Complaint.history`)
Implements an audit trail for transitions:
```typescript
interface ComplaintHistory {
  id: string;              // "hist_xxxxx"
  complaintId: string;     // Reference -> Complaint.id
  status: ComplaintStatus;
  priority?: ComplaintPriority;
  note?: string;           // Operator comments
  actorName: string;       // Authorized handler name
  actorRole: 'admin' | 'resident';
  timestamp: string;
}
```

## 📡 REST API Documentation

### 🔑 Authentication
- **`POST /api/auth/login`**
  - **Payload:** `{ "email": "john@society.com" }`
  - **Response:** `{ "user": { "id": "usr_resident1", "email": "john@society.com", "role": "resident" ... } }`

- **`POST /api/auth/register`**
  - **Payload:** `{ "email": "new@society.com", "name": "Jane", "role": "resident" }`

### 🔧 Complaint Management
- **`GET /api/complaints?userId=usr_resident1`**
  - Admins fetch all complaints (sorted overdue first). Residents fetch their own list.
- **`POST /api/complaints`**
  - **Payload:** `{ "residentId": "usr_resident1", "category": "Plumbing", "description": "Leaking Pipe", "photoUrl": "data:image...", "simulateDaysAgo": 0 }`
- **`PUT /api/complaints/:id`**
  - **Payload:** `{ "adminId": "usr_admin1", "status": "In Progress", "priority": "High", "note": "Technician scheduled" }`

### 📢 Notice Board
- **`GET /api/notices`**
  - Returns notices. Pinned notices are automatically sorted to the top.
- **`POST /api/notices`**
  - **Payload:** `{ "adminId": "usr_admin1", "title": "AGM Today", "content": "Dear residents...", "isImportant": true }`

### ✉️ Simulated SMTP Service
- **`GET /api/emails`**
  - Returns sent logs for visual inspection.
